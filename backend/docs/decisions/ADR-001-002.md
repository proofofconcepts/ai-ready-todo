# ADR 001 — Clean Architecture

**Data:** 2026-05-19 | **Status:** Aceito

## Contexto
Precisávamos de uma arquitetura testável, escalável e compreensível para assistentes de IA.

## Decisão
Adotamos Clean Architecture com quatro camadas:
1. domain — entidades, eventos, interfaces
2. application — use cases, DTOs
3. infrastructure — implementações concretas
4. interfaces — controllers HTTP

## Consequências
**Positivas:** testes sem banco, use cases isolados, padrão claro para o assistente de IA.
**Negativas:** mais arquivos por feature.

## ADR 002 — Storage em memória

**Data:** 2026-05-19 | **Status:** Aceito

## Contexto
Este é um projeto de referência educacional. Banco de dados adicionaria complexidade desnecessária.

## Decisão
Usar InMemoryTodoRepository — array JavaScript como storage.
Dados são perdidos ao reiniciar — comportamento esperado.

## Consequências
**Positivas:** zero configuração, foco nos conceitos AI-Ready.
**Negativas:** dados não persistem entre reinicializações.
Para produção: substituir InMemoryTodoRepository por TypeOrmTodoRepository sem mudar nenhum use case.
